# Mitra Central — React Migration Setup Guide

## What Changed

This update migrates **QAD-Zone** and **Apex** from Alpine.js to React 18
while keeping all other pages (Mitra, Visual Intelligence) on Alpine.js unchanged.

### New features delivered
| Feature | Where |
|---------|-------|
| File upload in QAD-Zone (Query + Docs modes) | `.p .i .xml .zip` support |
| Apex embed prompt after doc generation | "Yes, embed" → Qdrant `qad_custom_docs` |
| In-session module switcher in Apex | Dropdown in header, no page refresh |
| `custom_docs` domain in Apex | Queries embedded custom module docs |
| `POST /agents/qadzone/embed` API | Chunks + embeds Word doc to Qdrant |
| `POST /agents/apex/context` API | Updates domain filter mid-session |

## Step 1 — Install dependencies

```bash
cd frontend
npm install
```

This installs: React 18, Vite, @vitejs/plugin-react, lucide-react, marked.

---

## Step 2 — Build React bundles

```bash
# From the frontend/ directory:
npm run build
```

This outputs two files into `app/static/js/dist/`:
- `qadzone.bundle.js` — QAD-Zone React component
- `apex.bundle.js`    — Apex React widget

> **You must run this build every time you change a frontend file.**
> The FastAPI server serves the built files as static assets.

---

## Step 3 — Update .env

Add this line to your `.env` file:

```env
QDRANT_COLLECTION_CUSTOM=qad_custom_docs
```

---

## Step 4 — Run the server

```bash
# From project root:
uvicorn app.main:app --reload
```

Everything else (auth, Mitra, Visual Intelligence) works exactly as before.

---

## Development workflow

For live React development, run Vite dev server alongside FastAPI:

```bash
# Terminal 1 — FastAPI
uvicorn app.main:app --reload --port 8000

# Terminal 2 — Vite HMR dev server
cd frontend
npm run dev
```

Then visit `http://localhost:5173` — Vite proxies API calls to FastAPI on 8000.

> Note: In dev mode, the browser hits Vite (port 5173) for React HMR.
> In production, just `npm run build` and serve FastAPI normally on port 8000.

## How file upload works (QAD-Zone)

```
User uploads .p / .i / .xml / .zip
          ↓
FileReader → base64 encode in browser
          ↓
WS payload: { mode, question, uploaded_files: [{name, data}] }
          ↓
Backend service.py:
  _extract_uploaded_code() → decode base64 → extract ZIP if needed
  → concatenate all file contents as code string
          ↓
Code string used as LLM context (replaces disk module lookup)
```

## How Apex embed works

```
QAD-Zone generates a Word doc
          ↓
DocCard shows "Embed in Apex?" prompt
          ↓
User clicks "Yes, embed"
          ↓
POST /agents/qadzone/embed { doc_url, title }
          ↓
embedder.py:
  1. Opens the .docx from app/static/downloads/
  2. Extracts text chunks by heading section
  3. Embeds each chunk via OpenAI text-embedding-3-large
  4. Upserts to Qdrant collection "qad_custom_docs"
     with metadata: { module, section, doc_title, source }
          ↓
Apex widget: select "Custom Modules" in dropdown
          ↓
Apex queries qad_custom_docs collection → precise answers about your code
```

## How in-session module switching works (Apex)

```
User opens Apex → picks initial domains (e.g. "sales")
          ↓
Header shows ModuleDropdown with current active domains
          ↓
User clicks dropdown → selects/deselects domains
          ↓
POST /agents/apex/context { domains: ["sales", "custom_docs"] }
  → saves to in-memory session store
          ↓
Next message automatically uses the new domain filter
No page refresh, no session reset, conversation history preserved
```

## Qdrant collections

| Collection | Content | Used by |
|-----------|---------|---------|
| `qad_docs` (or `apex-normal`) | Official QAD user guides | Apex (standard domains) |
| `qad_custom_docs` | Embedded QAD-Zone Word docs | Apex ("Custom Modules" domain) |

The `qad_custom_docs` collection is auto-created on first embed with correct
vector dimensions and a keyword index on the `module` field.

## Troubleshooting

**React components not loading:**
- Run `npm run build` in `frontend/` first
- Check `app/static/js/dist/` exists and contains the bundle files
- Check browser console for JS errors

**"Embed failed" in DocCard:**
- Verify Qdrant URL and API key in `.env`
- Check `app/static/downloads/` contains the referenced `.docx` file
- Check FastAPI logs for the specific error

**Module dropdown not updating search:**
- Check browser Network tab for `POST /agents/apex/context` response
- Verify the session cookie is being sent (`credentials: "same-origin"`)

**Tailwind classes not applying in React:**
- All styling uses the CDN Tailwind loaded in `base.html` — no build step needed
- The Tailwind CDN scans the page at runtime, so all `className` strings work
